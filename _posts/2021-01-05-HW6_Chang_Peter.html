Within the training dataset, perform 5-folds cross validation to select the best tuning parameter \(K\). First pre-specify the range of \(K\) to be odd number: \(K = \{1,3,5,\ldots, 29\}\). Note that if multiple \(K\) achieve the same cross validation error rate, choose the smallest \(K\) since the classifier is parsimonious. (Another note: here \(K\) refers to the number of neighbours in the KNN algorithm. \(K\) is not the number of data splitting in the cross-validation. For the cross-validation, number of data splitting is set to be 5.)

## your code
K <- seq(1, 29, 2)

fold <- 5
folds <- cut(seq(1,nrow(prostate_train_x)),breaks=5,labels=FALSE)

EKs <- numeric(fold)
pre <- numeric(length(K))

for (i in K) {
  for (f in 1:fold) {
    #first fold the training data set
    atrain1 <- subset(prostate_train_x, folds != f)
    atest1 <- subset(prostate_train_x, folds == f)
    
    alabel_train <- subset(prostate_train_y, folds != f)
    alabel_test <- subset(prostate_train_y, folds == f)
    
    #then fit via KNN
    knnFit <- knn(atrain1, atest1, alabel_train, k=i)
    
    knnTable <- table(knnFit, truth=alabel_test)
    
    #get prediction rates from folds
    EKs[f] <- 1 - sum(diag(knnTable))/sum(knnTable)
    
    #gets final prediction rates via mean
    pre[i] <- mean(EKs)
  }
}

The underlying truth is 0.0013499, which can be estimated by \(P(Y>3) = \int_3^\infty \frac{1}{2\pi} \exp(-\frac{t^2}{2}) dt\). In fact, \[P(Y>3) = \int_3^\infty \frac{1}{\sqrt{2\pi}} \exp(-\frac{t^2}{2}) dt = \int_{-\infty}^\infty h(t)\frac{1}{\sqrt{2\pi}} \exp(-\frac{t^2}{2}) dt,\] where \(h(t) = 1\) if \(t>3\) and \(h(t) = 0\) if \(t\ \le 3\).

Assume we have one observation \(X \sim Binomial(n, \theta)\) (i.e.\(f(x | \theta) = \binom{n}{x}\theta^{x} (1 - \theta)^{n-x}\)). The prior for \(\theta\) is \(\pi(\theta) \sim Beta(a,b)\). By Bayes rules, we can derive \[p(\theta|x) \propto f(x | \theta) \pi(\theta) = \theta^{x} (1 - \theta)^{n-x} \theta^{a - 1} (1 - \theta)^{b - 1} = \theta^{x+a-1} (1 - \theta)^{n-x+b-1}\] So the posterior density function for \(\theta\) \(p(\theta|x) \sim Beta(x+a,b+n-x)\).

Question D4. implement MH algorithm (5 pts)

We still draw x from \(X \sim Binomial(n, \theta)\) (i.e.\(f(x | \theta) = \binom{n}{x}\theta^{x} (1 - \theta)^{n-x}\)). For \(\theta\), instead of directly draw \(\theta\) from the posterior density function (i.e., \(p(\theta|x) \sim Beta(x+a,b+n-x)\)), we will using MH algorithm to get samples. Suppose the sampler distribution is Gaussian distribution \(\theta' \sim N(\theta, 0.2^2)\), where \(\theta\) is current step value and \(\theta'\) is the next step value. Again initialize \(\theta = 0.5\), and \(X = 5\). Set \(a=b=2\), \(n=10\). Repeat MH algorithm \(B = 1,000\) times to iteratively update \(\theta\) and \(x\). Visualize the time series plot of \(\theta\). What is the percentage of times a new proposed \(\theta\) is accepted?

## your code
set.seed(123)

#prior parameters
a = 2
b = 2
n = 10

#target distribution
p <- function(y) {((y)^(x+a-1))*((1-y)^(n-x+b-1))}

#starting values
x = 5
theta = .5

#sampler starting value
std <- .2^2

B <- 1000
accepted <- 0 #accepted proposals
u <- runif(B)

draws2 = matrix(0, B, 2)
draws2[1,] = c(x, theta)
colnames(draws2) =  c('x','theta')

for (i in 2:B) {
  #drawing theta
  draws2[i, 1] <- rbinom(1, size = n, prob = theta)
  
  thetap <- rnorm(1, mean = theta, sd = std)
  
  alpha <- min(1, 
           (p(thetap)/p(theta))*(dnorm(theta, thetap, sd = 1)/dnorm(thetap, theta, sd = 1)))
  
  draws2[i, 2] <- theta <- ifelse(u[i] < alpha, thetap, theta)
  
  accepted <- accepted + (u[i] < alpha)
} 


accepted/B #percentage when the proposed theta was accepted
## [1] 0.901
ts.plot(draws2[,2], xlab = 'Samples', ylab = '', main = 'theta')
